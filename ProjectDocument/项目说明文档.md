# 基于 React 的塔罗牌占卜 Web 应用设计与实现

## 1. 项目概述

本项目拟设计并实现一个基于 React 与 Next.js 的沉浸式塔罗牌占卜 Web 应用。系统以“深色古籍女巫桌面”为核心视觉风格，通过黑屏显现、烛光占卜台、鼠标轨迹洗牌、拖拽抽牌、三维翻牌和 AI 流式解读等交互方式，为用户提供一次具有仪式感的塔罗占卜体验。

项目不是单纯的随机抽卡程序，而是一个结合前端交互动效、塔罗牌数据管理、本地历史记录和大语言模型解读能力的综合型 WebApp。用户可以输入自己的问题，完成洗牌与抽牌流程，并获得由“古老女巫”角色生成的过去、现在、未来三牌解读。

## 2. 项目名称

基于 React 的塔罗牌占卜 Web 应用设计与实现

## 3. 核心定位

一个结合古典视觉美学、物理交互动效与大语言模型解读能力的沉浸式塔罗占卜 WebApp。

## 4. 设计目标

1. 实现完整的塔罗占卜流程，包括提问、洗牌、抽牌、翻牌、解读和保存记录。
2. 构建具有强视觉记忆点的“深色古籍女巫桌面”界面风格。
3. 使用 Framer Motion 实现镜头变化、卡牌散落、拖拽抽牌和 3D 翻牌动画。
4. 通过 Next.js API Route 接入 DeepSeek API，隐藏密钥并完成 AI 解读。
5. 使用 localStorage 保存本地占卜历史，避免登录系统带来的额外复杂度。
6. 形成适合毕业设计展示、答辩和论文撰写的完整系统结构。

## 5. 目标用户

本项目面向对塔罗、占卜、情绪探索和沉浸式互动体验感兴趣的普通用户。用户不需要专业塔罗知识，只需要输入自己的问题，并按照界面引导完成一次占卜仪式。

## 6. 核心体验流程

系统主流程如下：

1. 页面初始为黑屏状态。
2. 画面中央逐渐显现烛光占卜台。
3. 首页标题、引导语和羊皮纸问题输入浮层逐渐出现。
4. 用户输入问题后点击“开始仪式”。
5. 镜头上抬，视角切换为垂直俯视桌面。
6. 中央牌堆进入洗牌阶段。
7. 用户通过鼠标移动或拖拽轨迹扰动牌堆，卡牌产生散开、旋转、错位和层级交叠。
8. 用户点击“命运已乱，开始抽牌”结束洗牌。
9. 散乱卡牌自动飞回页面底部，形成横向弧形扇面牌带。
10. 用户从底部牌带中拖拽三张牌，依次放置到“过去 / 现在 / 未来”位置。
11. 前端将用户问题、三张牌、正逆位和牌阵位置提交给后端 API。
12. 后端一次性请求 DeepSeek API 生成完整三牌解读。
13. 前端按“过去 / 现在 / 未来”逐张翻牌，并分段流式展示 AI 解读。
14. 三张牌全部揭示后，未选中卡牌飞出屏幕并销毁。
15. 页面仅保留三张命运主牌、完整解读和“命运手记”。
16. 用户可选择保存本次占卜记录到本地历史。

## 7. 功能模块划分

### 7.1 首页仪式模块

首页采用“烛光占卜台式”设计。页面从黑屏开始，中央桌面区域逐渐显现。标题、引导语和问题输入区带有渐入和轻微模糊转清晰效果。

首页输入区采用羊皮纸浮层，而非现代玻璃拟态。浮层应呈现旧纸纹理、暗角、墨迹边缘和烛光阴影。按钮采用黄铜铭牌、蜡封或古书书签样式。

### 7.2 洗牌交互模块

用户点击“开始仪式”后，首页文本与输入浮层淡出，镜头从略带斜视角的烛光桌面上抬为垂直俯视视角。中央牌堆进入可交互状态。

洗牌阶段中，用户的鼠标移动轨迹会对牌堆产生扰动。卡牌应根据鼠标轨迹发生不规则散开、旋转、错位和层级变化，模拟用手在桌面上抹开牌堆的感觉。

### 7.3 抽牌牌阵模块

用户点击“命运已乱，开始抽牌”后，散乱卡牌自动收束到底部，并排列成横向弧形扇面牌带。该排列方式仅参考 Moonlight 示例中的底部扇形结构，不参考其浅色视觉风格。

页面中央出现三个放置区域，分别为：

- 过去
- 现在
- 未来

用户需要从底部牌带中拖拽三张牌，并依次放置到三个命运位置。每成功放置一张牌，该牌从底部牌带移除，并以背面状态固定在对应位置。

### 7.4 翻牌解读模块

三张牌全部放置完成后，系统进入逐张翻牌阶段。翻牌顺序固定为：

1. 过去
2. 现在
3. 未来

每次只聚焦一张牌。当前牌轻微上浮并放大，背景压暗，卡牌执行 3D 翻转动画，从背面翻至正面。对应的 AI 解读文本以打字机或流式效果逐字显示。

### 7.5 终局手记模块

当三张牌全部揭示并完成解读后，系统进入终局清场阶段。所有未被选中的底部散牌以弹簧动画飞出屏幕边缘并销毁，画面仅保留三张命运主牌。

系统生成“命运手记”，内容包括：

- 用户问题
- 抽牌时间
- 三张牌名称
- 正逆位状态
- 过去 / 现在 / 未来位置
- AI 分段解读
- AI 最终总结箴言

用户可选择保存本次占卜记录。

### 7.6 数据与管理模块

系统不设计用户登录和账号体系。占卜历史记录保存在浏览器 localStorage 中。该策略可以降低开发复杂度，使毕业设计重点集中在视觉交互、塔罗数据结构和 AI 解读流程上。

系统内置完整 78 张塔罗牌数据，包括 22 张大阿卡纳和 56 张小阿卡纳。牌库初期使用 TypeScript 本地数据文件维护。

## 8. 视觉风格规范

### 8.1 视觉基调

项目采用“深色古籍女巫桌面”风格，整体氛围偏神秘、复古、克制。界面模拟用户坐在女巫占卜桌前进行塔罗仪式的体验，强调暗色空间、旧木桌面、烛光阴影、羊皮纸和黄铜光泽。

### 8.2 色彩系统

主色调：

- 近黑色
- 黑褐色
- 深木棕
- 暗酒红

辅助色：

- 旧羊皮纸色
- 黄铜金
- 暗墨绿
- 烛光琥珀色

高亮元素避免霓虹感，优先使用柔和烛光、低饱和金色光晕和纸张反光。

### 8.3 材质语言

核心材质包括：

- 旧木桌面
- 羊皮纸
- 古籍书页
- 黄铜边框
- 墨迹
- 蜡封
- 手写批注
- 轻颗粒纹理

星象元素仅作为装饰纹样、牌背符号或边框印章出现，不使用大面积星空背景。

### 8.4 牌面资源策略

项目采用完整 78 张 Rider-Waite-Smith 真实塔罗牌面，优先使用 Wikimedia Commons 中同一套 Public Domain 图像资源，避免混用现代商业复刻版本。

前端展示时不直接裸用原图，而是为牌面增加统一的旧牌外框、暗金描边、阴影和轻微复古滤镜，使真实塔罗牌面与“旧女巫木桌”整体风格融合。卡背不使用原始牌组卡背，而是使用项目自定义的 Sharp Gothic Ritual 卡背。

## 9. 技术架构

### 9.1 前端框架

使用 React 与 Next.js App Router 架构。普通展示页面优先使用 Server Components，涉及状态、拖拽、动画和浏览器 API 的组件使用 Client Components，并在文件顶部标注 `"use client"`。

### 9.2 样式方案

使用 Tailwind CSS 实现页面布局、响应式设计、颜色系统和基础 UI 样式。

### 9.3 动画方案

使用 Framer Motion 实现：

- 页面黑屏显现
- 镜头上抬
- 卡牌散落
- 弹簧收束
- 弧形牌带交互
- 拖拽抽牌
- 3D 翻牌
- 终局清场

### 9.4 AI 接口

使用 DeepSeek API 作为 AI 解读能力来源。DeepSeek API 由 Next.js API Route 在服务端调用，前端不直接暴露 API Key。

### 9.5 本地存储

使用 localStorage 保存占卜历史记录。当前版本不实现用户登录、鉴权和数据库存储。

## 10. 数据结构概述

### 10.1 塔罗牌数据

每张塔罗牌应包含：

- 唯一 ID
- 中文名
- 英文名
- 所属类型
- 花色
- 编号
- 正位关键词
- 逆位关键词
- 简短牌义
- 牌面图片路径

### 10.2 占卜记录数据

每条占卜记录应包含：

- 记录 ID
- 用户问题
- 创建时间
- 三张牌信息
- 每张牌的位置
- 每张牌的正逆位
- AI 分段解读
- AI 总结箴言

## 11. AI 人设与文本风格

AI 角色设定为“古老女巫”。解读文本应呈现神秘、晦涩、富有诗意与宿命论的语调。

示例语气：

> 火光已经听见你的疑问。旧牌不会给出答案，它只揭开你早已走向的影子。

占位符文本不得使用 Lorem Ipsum，应使用符合项目氛围的中文测试句。

## 12. 项目创新点

1. 将传统塔罗占卜流程转化为完整的沉浸式 Web 交互体验。
