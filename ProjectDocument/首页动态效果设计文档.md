# 首页动态效果设计文档

## 1. 文档目的

本文档用于记录塔罗牌 WebApp 首页与相关阶段的动态效果设计方案。后续所有首页动效、环境光、卡牌入场、洗牌衔接和阅读揭示效果的代码实现，都应以本文档作为设计依据，并在方案变化时同步更新本文档。

## 2. 整体动效目标

首页动效服务于“旧女巫木桌风”的沉浸式仪式感。动效应明显可见，但不能破坏深色古籍、烛光、羊皮纸和真实塔罗牌面的克制氛围。

设计原则：

- 背景图保持当前构图，不重新生成背景，不新增真实火焰主体。
- 使用前端叠加 CSS / Framer Motion 光效模拟烛光、桌面暖光和少量尘埃。
- 避免现代霓虹、科技粒子、大面积星空、全屏魔法阵和高饱和特效。
- 3D 效果重点放在卡牌本体，不对 2D 背景做明显 3D 旋转。
- 首页牌堆从一开始就是后续洗牌流程的真实牌堆组件，而不是装饰预览。

## 3. 页面与牌堆阶段

页面主流程建议调整为：

```ts
type AppStage =
  | "intro"
  | "question"
  | "ritual-transition"
  | "shuffle"
  | "fan"
  | "draw"
  | "reading"
  | "final";
```

牌堆视觉阶段建议单独管理：

```ts
type DeckVisualStage =
  | "home-stack"
  | "shuffle-ready"
  | "shuffle-active"
  | "fan"
  | "draw";
```

阶段说明：

- `home-stack`：首页整齐压紧的一摞真实牌堆。
- `shuffle-ready`：标题和羊皮纸退场后，牌堆留在原位，等待鼠标经过触发洗牌。
- `shuffle-active`：鼠标轨迹开始扰动牌堆，卡牌逐步散开。
- `fan`：散乱牌收束到底部弧形牌带。
- `draw`：用户从底部牌带拖拽三张牌到过去、现在、未来位置。

## 4. 环境光系统

环境光贯穿首页、洗牌、抽牌、阅读和终局阶段，但不同阶段强度不同。

建议组件命名：

```txt
AmbientLightEffects
```

建议模式：

```ts
type AmbientMode =
  | "home"
  | "transition"
  | "shuffle"
  | "draw"
  | "reading"
  | "final";
```

环境光分层：

1. `candle-local-left`：左上蜡烛局部跳动光。
2. `candle-local-right`：右上蜡烛局部跳动光。
3. `candle-sweep`：两侧烛光向桌面中心延展形成的低强度扫光。
4. `center-breath`：中央桌面慢速暖光呼吸。
5. `candle-dust`：只在两侧烛光附近可见的少量尘埃。

当前实现状态：

- `src/components/AmbientLightEffects.tsx` 已实现阶段化环境光组件。
- `src/app/globals.css` 中已实现两侧蜡烛 flicker、中央暖光呼吸、烛光附近固定少量尘埃和 `prefers-reduced-motion` 降级。
- 当前阶段映射为：`intro/question -> home`、`camera-lift -> transition`、`shuffle/fan -> shuffle`、`draw -> draw`、`reading -> reading`、`final -> final`。
- 局部 flicker 光源中心已按当前截图对齐蜡烛火焰；光效元素可向界面外溢出，优先保证中心点与火焰一致；局部光晕尺寸已扩大。

背景图尺寸为 `1672 x 941`。参考光源位置：

- 左上蜡烛火焰中心：约 `10.8% / 3.2%`。
- 右上蜡烛火焰中心：约 `89.7% / 3.3%`。
- 中央桌面暖光：约 `50% / 45%`。

## 5. 环境光强度表

```txt
home:
candle 1.00 / center 1.00 / dust 1.00 / flicker 1.00

transition:
candle 1.05 / center 1.15 / dust 0.75 / flicker 1.00

shuffle:
candle 0.75 / center 0.85 / dust 0.35 / flicker 0.75

draw:
candle 0.65 / center 0.65 / dust 0.25 / flicker 0.55

reading:
candle 0.45 / center 0.50 / dust 0.10 / flicker 0.35

final:
candle 0.30 / center 0.35 / dust 0.00 / flicker 0.15
```

说明：

- 首页阶段环境光最完整，烛光 flicker、中央暖光和尘埃都明显。
- 洗牌与抽牌阶段保留氛围，但降低尘埃可见度，避免影响交互。
- 阅读阶段压低全局环境光，把视觉焦点交给当前翻牌。
- 终局手记阶段弱化 flicker，保证文字阅读。

## 6. 烛光与尘埃参数

烛光风格为真实蜡烛式快速细碎闪动，而不是慢速机械呼吸。

建议参数：

- 左上烛光周期：`1.6s - 2.4s`。
- 右上烛光周期：`1.3s - 2.1s`。
- 局部光透明度：`0.20 - 0.58`。
- 中央联动光透明度：`0.08 - 0.24`。
- 扫光层透明度：`0.05 - 0.18`。
- 局部光缩放：`0.92 - 1.10`。
- 中央光缩放：`0.98 - 1.04`。

局部 flicker 示例：

```txt
opacity: 0.32 -> 0.52 -> 0.38 -> 0.58 -> 0.44
scale:   0.96 -> 1.04 -> 0.98 -> 1.08 -> 1.00
```

尘埃规则：

- 尘埃不是全屏粒子背景。
- 主要分布在左上和右上蜡烛光晕范围内。
- 中央桌面几乎不可见，只允许少量进入扫光区域。
- 粒子大小：`1px - 2.5px`。
- 单侧数量：约 `8` 个。
- 透明度：`0.08 - 0.28`。
- 漂浮距离：`8px - 24px`。
- 周期：`7s - 14s`。

## 7. 首页入场时间轴

首页入场采用分层显现，节奏为“火光先醒，仪式再出现”。

```txt
0.0s - 0.4s
页面保持极暗，只能隐约看到桌面轮廓。

0.4s - 1.2s
左上、右上烛光开始快速细碎 flicker。
局部光晕先亮，尘埃只在烛光附近偶尔可见。

0.9s - 1.8s
中央桌面暖光被两侧烛光带亮。
桌面中心出现慢速呼吸光。

1.3s - 2.8s
标题单字乱序显现。

1.9s - 2.7s
真实 78 张牌堆以整齐压紧状态淡入。
牌堆边缘跟随烛光有轻微明暗变化。

2.4s - 3.2s
羊皮纸输入区淡入，阴影逐渐落到桌面上。
开始仪式按钮最后亮起。
```

## 8. 标题单字乱序显现

主标题使用单字拆分动效。标题字符最终仍按正常顺序排列，但出现顺序采用自定义顺序。

对四字标题，默认出现顺序：

```txt
第 3 字 -> 第 2 字 -> 第 1 字 -> 第 4 字
```

如果标题为“命运之牌”，显现顺序为：

```txt
之 -> 运 -> 命 -> 牌
```

最终显示仍为：

```txt
命运之牌
```

动效规则：

- 每个字不是等上一个完全结束后才出现。
- 下一个字在上一个字动画进入后半段时启动。
- 单字动画时长：`0.75s - 0.9s`。
- 字与字启动间隔：`0.22s - 0.28s`。
- 总标题显现时间：约 `1.5s - 1.8s`。
- 英文小标题和正文不使用单字乱序，只做整体淡入。

单字动画建议：

```txt
初始：
opacity: 0
filter: blur(12px)
y: 14px
scale: 0.96

中段：
opacity: 0.85
filter: blur(3px)
y: -2px
scale: 1.035
text-shadow: 烛金光稍强

结束：
opacity: 1
filter: blur(0px)
y: 0
scale: 1
text-shadow: 稳定低亮烛光
```

实现建议：

```ts
const title = "命运之牌";
const revealOrder = [2, 1, 0, 3];

function getDelay(index: number) {
  return revealOrder.indexOf(index) * 0.24;
}
```

渲染时仍按原字序排列，只根据 `revealOrder` 计算 delay。外层保留完整 `aria-label="命运之牌"`。

## 9. 统一牌堆组件方案

后续采用统一 `TarotDeck` 组件，不再让首页牌堆、洗牌牌堆和底部牌带成为互不相关的视觉对象。

技术决策：

- 首页 78 张真实牌堆已统一，当前首页和洗牌阶段复用同一批真实牌数据与同一个中轴锚点。
- 首页阶段渲染真实 78 张牌，但视觉上压成整齐牌堆。
- 点击开始仪式后，同一批牌从 `home-stack` 过渡到 `shuffle-ready`。
- 首页、仪式退场和洗牌阶段复用同一 DOM 牌组与中轴锚点，不通过两套牌堆交叉淡入，避免位置跳变和切换跳帧。
- 鼠标经过牌堆影响半径后，进入 `shuffle-active`。
- 现有鼠标扰动算法保留，并迁移到统一牌堆组件或配套 hook。
- 底部弧形牌带后续也由同一组件的 `fan/draw` 布局控制。

首页牌堆形态：

- 整齐压紧的一摞牌。
- 绝大多数牌几乎完全重叠。
- 顶部只露出轻微边缘错位，不能像扇形牌堆。
- 牌堆使用独立于动画层的中轴锚点固定在视口中心，避免动画 `transform` 破坏水平或垂直居中。
- 羊皮纸输入区使用底部安全距离定位并始终水平居中，不能与中央牌堆遮挡或重合；矮窗口下应压缩输入区纵向尺寸以维持安全间隔。
- 卡背尺寸建议约 `112px x 176px`。
- 总厚度错位：`8px - 14px`。
- 顶层旋转：`-1.2deg` 到 `1.2deg`。
- 外层整体呼吸：`scale 0.995 - 1.008`。

压缩厚度建议：

```ts
const depth = index / (deck.length - 1);
const y = depth * 12;
const rotate = topLayer ? smallTopJitter : depth * 0.15;
```

## 10. 仪式退场与洗牌衔接

点击“开始仪式”后，页面进入 `ritual-transition`。不做明显背景 3D 镜头变化，改为仪式转场。

退场时间轴：

```txt
0.0s - 0.35s
标题最先变暗淡出。
标题不是直接消失，而是烛光金逐渐失去亮度，轻微 blur 后退场。

0.12s - 0.65s
英文小标题、引导语、装饰分割线跟随淡出。

0.25s - 0.95s
羊皮纸输入区模糊下沉。
opacity 降低，filter blur 增强，y 向下 18px - 32px。
阴影同步变软，像纸片被暗处吞没。

0.45s - 1.10s
中央背景光收束到牌堆。
桌面整体稍微压暗，两侧蜡烛仍 flicker，但中央牌堆附近更亮。

0.75s - 1.20s
真实牌堆留在原位，不退场。
牌堆从 home-stack 切换为 shuffle-ready。
```

`shuffle-ready` 行为：

- 牌堆仍整齐，但中心光聚焦，提示可以开始扰动。
- 只要鼠标经过牌堆范围，就进入 `shuffle-active`。
- 第一次触发时不立刻爆散，顶层牌先轻微错位，再根据鼠标轨迹逐步散开。
- 不监听全屏无意义移动，只计算牌堆附近区域。

建议参数：

- 牌堆交互半径：`160px - 220px`。
- 激活阈值：鼠标进入半径且移动距离超过 `4px`。
- 顶层初始松动：前 `10 - 16` 张最明显。
- 近距离扰动：`x/y 20px - 80px`。
- 中距离扰动：`x/y 8px - 32px`。
- 远距离扰动：只改变 `rotate 1deg - 4deg`。

## 11. 阅读阶段布局修正

阅读阶段三张牌放置区需要整体上移，给底部讲解面板留出稳定空间，避免卡牌翻转后与解读组件重合。

布局规则：

- `draw` 阶段牌位可以位于视觉中心偏上，方便拖拽。
- `reading` 阶段牌位继续上移，给底部解读面板让出空间。
- 底部解读面板作为固定阅读区域，不被卡牌覆盖。
- 当前聚焦牌可以轻微放大和上浮，但最大边界不能侵入解读面板。
- 移动端需要单独降低卡牌尺寸，优先保证解读文本可读。

建议参数：

```txt
draw 阶段三牌区：
top: 24%

reading 阶段三牌区：
top: 18% - 22%

当前聚焦牌上浮：
y: -12px 到 -24px

当前聚焦牌放大：
scale: 1.04 - 1.08

底部解读面板：
bottom: 6% - 8%
max-height: 28vh - 34vh
```

## 12. 阅读阶段揭示特效

阅读阶段使用独立的 `CardRevealEffects`，不要把当前牌揭示粒子绑到全局环境光组件里。

建议组件：

```tsx
<CardRevealEffects active={isCurrentCard} revealKey={position} />
```

特效分层：

1. `reading-dim-layer`：全屏暗化，保留桌面和其他两张牌的可见度。
2. `card-spotlight`：当前牌周围暖色椭圆聚光。
3. `card-edge-glow`：当前牌边缘烛金高光。
4. `reveal-particles`：围绕当前牌触发的神秘金粉粒子。

规则：

- 当前牌有明显暖光聚焦。
- 正逆位视觉状态在翻牌完成后保持不变，进入后续讲解或命运手记动画时不得自动旋转回正位。
- 其他区域压暗，但不完全黑掉。
- 当前牌边缘高光在翻转时最明显，翻完后降低到稳定亮度。
- 粒子只围绕当前牌边缘和聚光区出现。
- 粒子在每张牌揭示时触发一次，随后淡出，不常驻。

时间轴：

```txt
0.0s - 0.4s
边缘高光增强。

0.2s - 1.2s
金粉粒子从牌边缘附近浮起。

0.3s - 0.9s
卡牌完成 3D 翻转。

1.0s 后
粒子淡出，只留下稳定边缘光。
```

金粉粒子参数：

```txt
数量：14 - 22 个
大小：1px - 3px
持续：1.2s - 2.0s
透明度峰值：0.35 - 0.75
扩散半径：当前牌外 18px - 70px
颜色：rgba(213,155,76), rgba(244,217,158), rgba(176,138,74)
```

视觉限制：

- 风格为神秘金粉，不是现代闪亮特效。
- 不做全屏粒子。
- 不做爆炸式粒子。
- 不使用霓虹紫蓝。
- 不使用现代魔法阵。

## 13. 性能与可访问性约束

- 优先使用 `transform`、`opacity` 和少量 `filter`。
- 不对 78 张牌同时应用高成本 blur 或复杂阴影。
- 环境光可以是少量大面积绝对定位层，不对每张牌单独计算光照。
- 粒子数量必须克制，首页尘埃和阅读金粉都不应全屏铺满。
- 背景不做明显 `rotateX`，避免 2D 背景透视破绽。
- 需要兼容 `prefers-reduced-motion`，在减少动态模式下降低 flicker、粒子和位移。
