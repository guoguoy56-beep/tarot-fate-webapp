# 塔罗牌 WebApp 美术风格规范文档

## 1. 风格定位

本项目整体美术方向确定为 **旧女巫木桌风**。界面应呈现用户坐在昏暗木桌前进行塔罗占卜仪式的感觉，强调旧木桌、烛光、羊皮纸、黄铜边框、哥特仪式感字体和真实塔罗牌面之间的统一。

关键词：

- 旧女巫桌面
- 古典手帐
- 烛光仪式
- 哥特塔罗
- 羊皮纸札记
- 暗金细线
- 复古真实牌面

整体风格应神秘、古典、克制，不使用现代霓虹、科技星空、大面积玻璃拟态或过度商业化的明亮渐变。

## 2. 首页整体视觉

首页为项目第一视觉记忆点，采用完整的旧女巫桌面场景：

1. 背景为柔和绘制感旧木桌。
2. 画面中央保留暖金色烛光晕染。
3. 四周明显压暗，形成暗角和沉浸感。
4. 标题、牌堆和输入浮层沿中轴排布。
5. 桌面边角可摆放少量装饰道具，但不得干扰中心交互区域。

首页需要避免牌堆与输入框重合。牌堆应作为独立的中心交互物存在，和羊皮纸输入浮层之间保留清晰间距。

## 3. 背景素材规范

背景选择 **柔和绘制感旧木桌背景**。

该背景贯穿首页、洗牌、抽牌、阅读和终局完整流程，不在仪式开始后切换回早期 DEMO 桌面。后续阶段保持与首页相同的背景遮罩和可见范围，不额外加强暗角；允许使用轻微放大或缩小作为镜头过渡。

规范如下：

- 非照片写实，而是带轻微绘制感的暗色旧木桌。
- 木纹低对比，不抢标题、牌堆和输入区。
- 中央有柔和琥珀色烛光晕染。
- 边缘暗角明显，形成仪式空间。
- 桌面可出现轻微灰尘颗粒和旧纸阴影。
- 不使用大面积星空、现代粒子背景或高饱和渐变。

## 4. 桌面装饰道具

桌面允许出现少量道具，用于强化旧女巫桌面氛围。

推荐道具：

- 小蜡烛
- 旧笔记本或咒文书
- 羽毛笔
- 墨水瓶
- 零散羊皮纸片
- 干花
- 黄铜小饰物
- 月相或圆环小符牌

摆放规则：

- 道具只放在桌面边缘或角落。
- 不遮挡标题、牌堆、输入框和按钮。
- 不作为主要交互控件。
- 明度低于中心 UI，避免喧宾夺主。
- 风格应绘制化、复古化，避免真实照片贴图感。

## 5. 字体与文字气质

主标题采用 **Sharp Ritual Gothic Serif** 风格。

### 5.1 中文主标题

- 高对比 serif 字形。
- 笔画具有尖锐收笔和仪式感。
- 气质接近古老仪式书、女巫咒文、哥特手抄本。
- 颜色使用羊皮纸金或烛光金。
- 避免现代黑体、可爱手写体和过于圆润的字体。
- 当前首页主标题采用用户选定的左上角方向：暗金雕刻感字符级标题，四字大小有明显但克制的差异，`命` 最大、`运` 略小、`之` 缩小夹在中间、`牌` 高挑收尾；标题上下使用火焰、月相、细线和星芒作为克制仪式装饰。标题颜色应偏旧铜金和暗金，不使用过浅羊皮纸金；字面内部保持干净的雕刻高光，不叠加明显条纹。

### 5.2 英文小标题

- 使用大写 Serif。
- 字距拉开。
- 作为仪式标签出现，例如 `THE FATE CARDS`、`OLD WITCH TABLE`。

### 5.3 正文与输入文字

- 正文保持可读性。
- 可使用宋体、仿宋或古典 serif 气质字体。
- 不与主标题一样夸张，避免影响阅读。

## 6. 色彩系统

### 6.1 主色

- 近黑色
- 黑褐色
- 深木棕
- 暗酒红

### 6.2 辅助色

- 羊皮纸黄
- 黄铜金
- 烛光琥珀
- 暗墨棕

### 6.3 使用原则

- 高亮元素使用低饱和烛金，不使用纯白高亮。
- 按钮、边框、卡背线条统一使用暗金或黄铜金。
- 大面积背景保持暗色，避免页面发灰或发亮。

## 7. 卡背设计规范

卡背方向确定为 **Sharp Gothic Ritual 卡背**。

当前正式卡背采用已选定的 **Eclipse Reliquary / 日蚀圣龛** 方案，素材文件为 `public/assets/tarot-card-back.webp`。中心图形由尖拱圣龛、日蚀圆环和小型烛火组成，使用近黑棕旧皮革底色与磨损暗金线稿；首页牌堆、洗牌、底部牌带、抽牌和翻牌背面统一使用该素材。

规范如下：

- 底色为近黑棕，带旧纸或旧皮革颗粒质感。
- 主图形为烛金色几何圆环、火焰、月相或仪式符号。
- 边框采用尖锐、克制的哥特仪式线条。
- 复杂度中等偏低，保证牌堆叠放和动画时仍然清晰。
- 不使用真实牌组卡背，不使用现代魔法阵和高饱和星空图案。
- 网页成品尺寸为 `704 x 1152` WebP，按约 `11:18` 的卡牌比例裁切和压缩。
- 前端卡背外层边缘使用深黑棕暗铜色，接近卡背近黑棕主色但略有区分；避免使用偏亮黄褐描边，以免 78 张首页牌堆叠后出现突兀亮边。

## 8. 真实牌面素材规范

项目采用完整 **78 张 Rider-Waite-Smith 真实塔罗牌面**。

### 8.1 素材来源

- 当前使用 `metabismuth/tarot-json` release `cards.zip` 中的同一套 Rider-Waite-Smith 牌面 JPG 素材；该项目 README 标注 Rider-Waite-Smith 原始牌组为 Public Domain，仓库代码为 MIT。
- 早期优先考虑 Wikimedia Commons 中同一套 Public Domain Rider-Waite-Smith 图像资源；实际批量下载时触发限流，因此改用上述稳定素材包。
- 不混用不同扫描版本。
- 不直接使用现代商业复刻牌组图片。
- 文档中需保留素材来源说明。

### 8.2 文件处理

- 当前文件保存为 JPG，目录为 `public/cards/rws/`；后续如引入可靠图片转换工具，可统一转为 WebP。
- 统一尺寸比例和命名规则。
- 建议目录：`public/cards/rws/`。
- 建议命名示例：
  - `major-00-fool.webp`
  - `major-01-magician.webp`
  - `cups-01-ace.webp`
  - `swords-10-ten.webp`

### 8.3 前端融合

前端展示真实牌面时，应增加项目统一视觉包装：

- 旧牌外框
- 暗金描边
- 偏黄色老旧纸面滤镜
- 旧纸颗粒、暗角和磨损边缘
- 阴影
- 低饱和复古滤镜

真实牌面可以向内轻微缩进，使卡牌边缘更自然，但不得形成厚黑边框或最外层白色描边。当前缩进为 `5px`，若缩进后露出组件底层，底层使用 `#FCF8FD` 填充。前端在整个卡牌组件上层统一叠加 medium 强度的黄色老旧滤镜，并继续叠加轻中度暗角、角落褐色旧化、旧纸颗粒和贴着牌图内缘的褐色磨损渐变。牌面上不叠加项目自定义的中文牌名、英文牌名、关键词或正逆位文字，避免遮挡原始牌面。牌名、位置和正逆位信息只在牌下方说明区、阅读面板或命运手记文本区域展示。

卡背仍使用项目自定义的 Sharp Gothic Ritual 卡背。

### 8.4 正逆位显示

正逆位由抽牌结果决定。终局命运手记阶段不额外触发翻牌动画，也不额外旋转牌面，应保持用户抽出并翻开时的视觉状态。

## 9. 羊皮纸输入浮层

输入浮层确定为 **Journal Paper Slip / 手帐纸片感羊皮纸**。

规范如下：

- 像一张放在旧木桌上的手帐纸片。
- 颜色为暖羊皮纸黄。
- 边缘允许轻微不规则、磨损和自然阴影。
- 可有轻微折痕、纸纹和角落小型仪式符号。
- 装饰克制，保证输入文字和按钮可读。
- 不做强烈烧边、红蜡封或大面积污渍。

## 10. 按钮规范

### 10.1 开始仪式按钮

采用旧铜发光按钮：

- 暗铜或深棕底色。
- 细黄铜金边框。
- 边缘有轻微烛金发光。
- 文字小号、宽字距、serif 气质。
- 可搭配星芒或仪式小符号。
- hover 时边缘光感增强。

### 10.2 历史记录按钮

历史记录按钮同样采用边缘发光样式：

- 右下角悬浮。
- 暗色底，细金边。
- 视觉层级低于开始按钮。
- 图标可使用回旋箭头、旧时钟或手记符号。

## 11. 装饰元素规范

允许使用以下装饰元素：

- 火焰符号
- 星芒
- 月相
- 圆环
- 手帐角标
- 细金分割线

使用规则：

- 装饰元素服务于信息层级，不单独堆叠。
- 同一屏幕内装饰元素数量要克制。
- 线条统一为暗金或黄铜色。
- 避免水晶球、骷髅、魔杖等过于具象且风格复杂的道具。

## 12. 首页动效美术规范

首页动效应强调仪式开始前的缓慢显现：

1. 页面从黑屏或极暗桌面开始。
2. 中央烛光缓慢亮起。
3. 标题和引导语由模糊转清晰。
4. 牌堆轻微浮动或呼吸，不应剧烈晃动。
5. 羊皮纸输入浮层柔和淡入。
6. 点击“开始仪式”后，输入浮层高斯模糊渐隐，卡牌进入交互状态。

动效曲线优先使用快入缓停或自然阻尼曲线，避免机械线性动画。

## 13. 当前已确认素材方向

- 首页背景：柔和绘制感旧木桌背景。
- 主标题：Sharp Ritual Gothic Serif，首页已落地暗金雕刻感字符级标题与火焰、月相、细线装饰。
- 卡背：Sharp Gothic Ritual 卡背。
- 输入浮层：Journal Paper Slip 手帐纸片感羊皮纸。
- 按钮：旧铜细金边框发光按钮。
- 牌面：完整 78 张 Rider-Waite-Smith 真实牌面。
- 桌面道具：蜡烛、旧笔记本、羽毛笔、墨水瓶、羊皮纸片、黄铜小饰物等边角装饰。

## 14. 前端替换原则

在前端实现时，应遵循以下原则：

1. 优先替换首页视觉，不一次性重构全部流程。
2. 背景、道具可先合成为一张背景图，降低开发复杂度。
3. 卡背可作为独立图片或 CSS 组件，后续用于牌堆、洗牌和底部牌带。
4. 真实牌面资源应分批接入，但最终目标是完整 78 张。
5. 首页牌堆和输入浮层必须避免重叠。
6. 每次实现前需先确认实现方案，再进行代码修改。
